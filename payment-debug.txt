微信支付沙箱环境使用方法
========================

1. 配置说明
   在 application.yml 中配置以下参数：
   
   wx:
     appid: your-appid  # 微信小程序AppID
     secret: your-secret  # 微信小程序AppSecret
     pay:
       sandbox: true  # 是否启用微信支付沙箱环境
       mch-id: your-mch-id  # 商户号
       api-key: your-api-key  # 商户API密钥
       sandbox-key: 12345678901234567890123456789012  # 沙箱密钥

2. 沙箱环境特点
   - 沙箱环境不会真实扣款
   - 支付接口地址自动切换为：https://api.mch.weixin.qq.com/sandboxnew/pay/unifiedorder
   - 使用沙箱专用密钥进行签名验证
   - 适合开发调试阶段使用

3. 切换正式环境
   将 wx.pay.sandbox 设置为 false 即可切换到正式支付环境

如何获取沙箱账号和密钥
======================

1. 登录微信支付商户平台
   访问：https://pay.weixin.qq.com
   使用商户号和密码登录

2. 获取沙箱密钥
   - 进入"账户中心" -> "API安全"
   - 点击"沙箱密钥"选项
   - 系统会自动生成32位沙箱密钥
   - 复制密钥并配置到 application.yml 的 wx.pay.sandbox-key

3. 沙箱商户号
   - 沙箱环境使用与正式环境相同的商户号
   - 配置到 application.yml 的 wx.pay.mch-id

小程序真机调试支付时注意事项
============================

1. 域名配置
   - 确保小程序后台已配置支付域名为可信域名
   - 正式环境需要 HTTPS 协议
   - 开发阶段可在开发者工具中勾选"不校验合法域名"

2. 支付授权
   - 小程序需要用户授权才能调用支付接口
   - 确保用户已登录并获取到 openid
   - JSAPI支付流程：
     a. 前端调用 wx.login() 获取 code
     b. 后端使用 code 调用微信 jscode2session 接口换取 openid
     c. 后端使用 openid 调用统一下单接口获取 prepay_id
     d. 后端生成支付参数返回给前端
     e. 前端调用 wx.requestPayment 发起支付

3. 模拟支付功能
   - 开发模式下，会员页面会显示"调试：模拟支付成功"按钮
   - 点击后直接更新会员到期时间，无需真实支付
   - 仅在 NODE_ENV=development 时显示

4. 真机调试步骤
   a. 使用微信开发者工具打开小程序项目
   b. 点击"真机调试"按钮
   c. 扫描二维码在手机上打开调试版本
   d. 进入会员页面进行支付测试

5. 常见问题
   - 支付失败：检查商户号、API密钥配置是否正确
   - 签名错误：确认使用的密钥类型（沙箱/正式）
   - 超时错误：检查网络连接和服务器响应时间
   - openid获取失败：检查AppID和AppSecret配置是否正确

6. 日志查看
   - 后端日志会显示支付参数和沙箱模式状态
   - 控制台输出示例：
     支付预下单 - 套餐: yearly, code: xxx, 沙箱模式: true
     获取openid成功: oXXXX...
     统一下单请求XML: <xml>...</xml>
     统一下单响应: <xml>...</xml>

7. 安全提示
   - 沙箱密钥仅用于测试，不要在生产环境使用
   - 正式上线前务必切换到正式环境并配置正确的API密钥
   - 定期更换API密钥以确保支付安全
   - AppSecret不要暴露在前端代码中，必须在后端使用
